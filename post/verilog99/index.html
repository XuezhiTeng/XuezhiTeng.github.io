Transition Time:
针对同一个信号，不论它是输入信号还是输出信号，transition time指的都是信号完成低-高（高-低）跳变所需要的时间。
Propagation Delay:
Propagation delay是一个门延迟的概念，一般是指输入信号应该逻辑门的处理之后反应在输出信号的最大延迟，与之对应的最小延迟是contamination delay。

4. 什么是原码，反码，补码，符号-数值码。以4bit为例，给出各自表示的数值范围。

原码：符号位+真值，最高位表示符号位。 以4bit为例：
[+3]原 = 0011
[-3]原 = 1011
表示范围：-7到+7
原码中0000和1000都表示0。

反码：正数的反码是它本身，负数的反码将原码除符号位外逐位取反。 以4bit为例：
[+3]原 = [0011]原 = [0011]反
[-3]原 = [1011]原 = [1100]反
表示范围：-7到+7
反码中0000和1111都表示0。

补码：正数的补码是它本身，负数的补码将原码除符号位外逐位取反再加1。 以4bit为例：
[+3]原 = [0011]原 = [0011]反 = [0011]补
[-3]原 = [1011]原 = [1100]反 = [1101]补
表示范围：-8到+7
补码中0的表示只有一种形式，即0000，1000表示-8。

以上是有符号数，对于无符号数来说都是来表示整数，其原码、反码、补码都是其本身。

更详细解释可参考维基百科

6. 画出CMOS三态缓冲器的电路原理图，解释一下高阻态。

先解释高阻态：高阻态一般用字母Z来表示，它既不是高电平也不是低电平，我理解为一种floating的状态。如果高阻态输入了下一级电路，对下一级电路是没有任何影响的。
在SoC中，一般使用总线来实现各个模块和CPU之间的交流，而挂在总线上的各个模块，在同一时刻只能有一个来驱动总线（不然的话就乱套了！）。所以其他不驱动的模块最好是输出高阻态。为了实现这一目的，常用的方法就是使用三态门（Tri-State）再配合一个使能信号。

三态门的真值表

ENENEN/EN‾\overline{EN}EN A Y
0/1 0 Z
0/1 1 Z
1/0 0 0
1/0 1 1

为实现上面的真值表最简单的方式就是Transmission Gate，但是TG也有响应的缺点：Nonrestoring. 也就是说输入端的噪声会不加处理地到达输出端，这对于数字电路的稳定性和准确性是不利的。另外几个TG连续出现而造成的延时是相当可观的。

另一个办法：Tristate Inverter.

补充说明：当我们使用三态门进行总线驱动控制时，没有被使能的三态门会处于一个floating状态（一个无效的逻辑电平），但依然浪费了能量；而且当芯片规模加大，很难使得不同的使能信号完全同步，一旦不同步则又会在总线上出现contention的情况，所以现在Deisgn Engineer更喜欢用MUX来解决问题。

8. 只用2输入mux，实现与，或，非，异或。

2输入mux定义为：
o = s ? a : b;

AND:
Y = A ? 0 : B;

OR:
Y = A ? B : 1;

NOT:
Y = A ? 1 : 0;

XOR:
Y = A ? B : ~B;
